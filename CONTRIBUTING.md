# Contributing to the Nava Open Source Guide

Thank you for your interest in contributing to the Nava Open Source Guide! This guide serves as Nava's central resource for open source practices and philosophy. We welcome contributions from Nava team members, government partners, and the broader open source community.

## Community

We are committed to providing a welcoming environment for all contributors. All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Bugs and Issues

If you find errors, broken links, or outdated information in this guide, please [create a GitHub issue](../../issues) using the bug report template. Search existing issues first to avoid duplicates.

If the issue is related to security, please email us directly at opensource@navapbc.com.

## Suggesting Content

Have an idea for a new guide section or topic? We'd love to hear it. Open a [feature request](../../issues/new?template=2_feature_request.yml) describing:

- What topic or question the content should address
- Who the intended audience is (Nava team members, external contributors, etc.)
- Any reference material or examples that could inform the content

## Getting Started

To contribute content, create a pull request on GitHub:

1. Fork the repository
2. Create a branch for your changes
3. Make your edits following the style guide below
4. Submit a pull request with a clear description of your changes

## Style Guide

When writing guide content:

- Use clear, direct language accessible to a broad audience
- Write in second person ("you") when addressing the reader
- Use headers to break up content for easy scanning
- Link to related sections within the guide where relevant
- Keep file and folder names lowercase with hyphens (e.g., `license-types.md`)

## Code Review Process

### Submitting for Review

- Ensure all links are valid
- Review your markdown formatting
- Self-review your changes for clarity and accuracy
- Request review from appropriate team members

### Review Criteria

Reviewers will check for:

- **Accuracy**: Is the information correct and up to date?
- **Clarity**: Is the content easy to understand?
- **Completeness**: Does it address the topic thoroughly?
- **Consistency**: Does it follow the guide's style and tone?
- **Navigation**: Are links and cross-references working?

### Addressing Feedback

- Respond promptly to review comments
- Make requested changes or discuss alternatives
- Re-request review after making changes

Thank you for helping improve the Nava Open Source Guide!
