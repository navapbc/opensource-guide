# Security Policy

## Supported Versions

We always recommend using the latest version of the Nava Open Source Guide to ensure you have the most current information.

## Reporting Vulnerabilities

Please do not file GitHub issues for security vulnerabilities, as they are public!

Nava takes security issues very seriously. If you have any concerns about the Nava Open Source Guide or believe you have uncovered a vulnerability, please get in touch via e-mail at opensource@navapbc.com. In the message, try to provide a description of the issue and ideally a way of reproducing it. The security team will get back to you as soon as possible.

Note that this security address should be used only for undisclosed vulnerabilities. Please report any security problems to us before disclosing it publicly.
