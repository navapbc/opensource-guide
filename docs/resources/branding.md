# Nava Open Source Branding

Consistent branding across Nava's open source projects helps people recognize our work and builds trust in the quality and reliability of our software.

## Repository Naming Conventions

- Use lowercase letters and hyphens (e.g., `benefit-delivery-systems`, not `BenefitDeliverySystems`)
- Choose descriptive names that communicate what the project does
- Avoid acronyms unless they're widely understood (e.g., `strata` is clear; `bds` is not)
- Keep names concise — aim for 1-3 words
- Don't include "nava" in the repository name (the organization name already provides that context)

## README Standards

Every Nava project README should include:

### Required Elements
- **Project name** as the H1 heading
- **One-line description** explaining what the project does
- **Longer description** (1-2 paragraphs) covering the problem it solves and who it's for
- **Getting started** instructions (prerequisites, installation, basic usage)
- **Contributing** link to CONTRIBUTING.md
- **License** statement

### Recommended Elements
- **Badges** for build status, license, and other relevant metrics
- **Architecture overview** for complex projects
- **Documentation links** if docs live elsewhere
- **Related projects** if part of a larger ecosystem

### Tone
- Professional but approachable
- Technical without being jargon-heavy
- Focus on what the project does for the user, not implementation details

## Nava Identity in Open Source

### When to Reference Nava
- Always include Nava's copyright in the LICENSE file
- Mention Nava in the README when it provides useful context (e.g., "Built by Nava PBC in partnership with [agency]")
- Use Nava's GitHub organization (navapbc) for all official projects

### When Not to Reference Nava
- Don't use Nava's logo in project READMEs without approval from the communications team
- Don't imply Nava endorses third-party projects
- Personal open source projects by Nava employees should not use Nava branding

### GitHub Organization Profile
The [navapbc GitHub organization](https://github.com/navapbc) is Nava's public face on GitHub:

- Keep the organization description and profile current
- Pin the most important and active repositories
- Use topics/tags consistently across repositories
