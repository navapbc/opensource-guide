# Resources and References

Open source touches licensing, copyright, branding, and government contract law — and getting these details right matters for the agencies we partner with and the people they serve. This section provides the foundational knowledge and reference material to help you navigate it all.

## Reference Guides

- [License Reference](licenses.md) — Nava's default license and how to choose the right one
- [Copyright Guidance](copyright.md) — Copyright at Nava and in government technology
- [Code Ownership](ownership.md) — Who owns the code and government contract implications
- [Branding Guidelines](branding.md) — Repository naming, README standards, and Nava identity

## Frequently Asked Questions

**Can I use open source software in a government project?**
Yes. See [Using Open Source](../using/index.md) for evaluation guidance and [License Types](../using/license-types.md) for compatibility details.

**Can I contribute to an external open source project?**
Yes. See [Contributing to Open Source](../contributing/index.md) for guidelines on how to do it well.

**How do I open source a Nava project?**
Follow the [Publishing](../publishing/index.md) process: prepare, get approval, publish, and manage releases.

**What license should I use?**
Apache 2.0 is Nava's default. See [License Reference](licenses.md) for details.

**Do I need permission to open source my work?**
Yes. See [Getting Approval](../publishing/approval.md) for the review process.

## Glossary

| Term | Definition |
|---|---|
| **ATO** | Authority to Operate — a government security authorization for a system |
| **CLA** | Contributor License Agreement — a legal agreement for project contributions |
| **Copyleft** | A license requiring derivative works to use the same license |
| **DCO** | Developer Certificate of Origin — a lightweight contribution agreement |
| **FOSS** | Free and Open Source Software |
| **OSI** | Open Source Initiative — the organization that maintains the Open Source Definition |
| **OSPO** | Open Source Program Office — an organizational unit managing open source strategy |
| **Permissive license** | A license with minimal restrictions on use and distribution |
| **SemVer** | Semantic Versioning — a versioning scheme using MAJOR.MINOR.PATCH |
| **SPDX** | Software Package Data Exchange — a standard for communicating license information |
